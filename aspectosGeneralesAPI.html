API(Application Programming Interface)

Una Interfaz de programación de aplicaciones API es un programa intermediario que está compuesto de rutinas, protocolos, herramientas y permite que dos o más aplicaciones de software se comuniquen entre sí.

Características

  • las API se las trata más como productos que como código.
  • Están dirigidas para el consumo de audiencias específicas por ejemplo desarrolladores Web.
  • Están documentados y tienen control de versiones
  • No hay que preocuparse de su mantenimiento y ciclo de vida.
  • No hay que preocuparse de los aspectos básicos de seguridad.
  • Hoy en día hay más de ciento sesenta mil de APIs disponibles en el mercado.

Popularización de las API:

El uso de las API se sitúa aproximadamente en el año 2000 con la entrada del comercio electrónico a través de la red mundial, cuando la empresa Salesforce.com incorporó las API en su plataforma para ayudar a los clientes a compartir y transmitir datos a través de sus diversas aplicaciones comerciales. Poco después, la empresa eBay comenzó a implementar esta tecnología y con la creación de las redes sociales unos años más tarde, empresas Facebook, Twitter e Instagram comenzaron a implementar el uso de las API para facilitar sus servicios.

Importancia de las APi representada mediante un caso hipotético:

Dentro del mundo actual del desarrollo de software las APIS son indispensables, pensemos un ejemplo que pasaría si una pequeña empresa de software quisiera ofrecer los servicios de Mapas haciendo el desarrollo por su propia cuenta pues le sería prácticamente imposible hacerlo ya que tendría que hacer un gran trabajo como hacer un barrido mundial de cartografía, ubicación de territorios, direcciones etc. Pero gracias a las API y específicamente a la API de Google Maps la empresa hipotética que mencionamos anteriormente puede hacer su desarrollo de Mapas y ofrecer sus servicios a un costo muy bajo comparado con el costo de desarrollar dicha tecnología por su propia cuenta.
